# Notik Reference Parity and CURA Superiority

## Purpose

Notik is a useful reference for fast recording, transcription, summaries, templates, learning outputs, content outputs, sharing, and export.

CURA should match the valuable capture-to-output behavior while establishing a clearer brand, deeper multimodal understanding, stronger Apple integration, stronger user ownership, and Mission Control-ready structure.

## Parity Targets

1. High-quality audio recording.
2. Pause and resume.
3. Audio and video transcription.
4. Public YouTube input where supported.
5. Smart summaries.
6. Translation.
7. Quick text notes.
8. Meeting follow-up.
9. Action points.
10. Email drafts.
11. Lecture notes.
12. Blog drafts.
13. To-do lists.
14. Quizzes.
15. Flashcards.
16. Share Sheet.
17. Text export.
18. PDF export.
19. Organized library.
20. Search.
21. Apple Watch as fast-follow if validated.

## CURA Superiority

1. Capture Session as the top-level object.
2. Multiple source types in one session.
3. Audio and visual understanding.
4. Unrelated-track detection.
5. Evidence-linked Curated Note.
6. Learn, Create, and Work.
7. Creator-ready Output Packs.
8. Property and operational workflows.
9. Visual Brief.
10. Quick Send.
11. Apple Shortcuts.
12. User-selected Files and iCloud export.
13. Local-first privacy.
14. Temporary processing by default.
15. Structured JSON.
16. Prompt and model versioning.
17. Mission Control-ready entities.
18. Clear brand and product personality.

## Boundary

CURA should not beat the reference by adding a full editor, scheduler, or dashboard to V1.

It should win through better capture, understanding, organization, outputs, trust, and activation.

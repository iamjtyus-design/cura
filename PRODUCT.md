# CURA Product Requirements Document

## Document Status

**Product:** CURA by VisionBuilt  
**Version:** 1.0  
**Stage:** Sellable beta planning  
**Primary platform:** iPhone  
**Future platforms:** Web dashboard, then Android  
**Primary source of truth:** `CURA_PRODUCT_CONSTITUTION.md`

This document defines what CURA must do, who it serves, how users experience it, and what qualifies as a successful beta.

When this document conflicts with the Product Constitution, the Constitution wins.

---

## 1. Executive Summary

CURA is a mobile-first, multimodal capture and knowledge activation platform.

CURA helps everyday people capture real-world information and quickly turn it into something useful.

Users can record or import:

1. Audio.
2. Video.
3. Photos.
4. Screenshots.
5. Typed notes.
6. PDFs.
7. Web pages.
8. YouTube links.
9. Supported social media links.
10. Files shared from other apps.

CURA then:

1. Transcribes spoken content.
2. Understands visual content.
3. Reads visible text.
4. Identifies ideas, decisions, tasks, people, dates, and themes.
5. Creates one organized Curated Note.
6. Generates useful outputs through Learn, Create, and Work modes.
7. Produces Visual Briefs.
8. Helps users quickly copy, export, save, share, or open another app.

CURA is a strong capture tool first.

It is not intended to become a professional editing suite, social scheduler, or full business dashboard.

---

## 2. Product Vision

CURA should become the default mobile tool people use when they think:

1. I need to capture this before I forget it.
2. I do not have time to organize this manually.
3. I need to understand this quickly.
4. I want to create something from this.
5. I need to remember what happened.
6. I want to move this into my notes, content system, or workflow.
7. I need the next action, not another raw transcript.

The long-term opportunity is to make CURA the trusted capture gateway into the VisionBuilt ecosystem.

## 5. Primary User

### Core audience

The everyday capturer.

This is someone who regularly encounters useful information and wants help turning it into something they can understand, remember, share, or act on.

### Primary behavioral profile

The user:

1. Uses their phone as the main capture device.
2. Records audio or video.
3. Saves screenshots and links.
4. Attends meetings, lectures, church, conferences, interviews, or site visits.
5. Wants fast outputs.
6. Does not want to manually clean up notes.
7. Uses several destination apps.
8. Values portability.
9. May not consider themselves technical.
10. Wants a clear result without learning a complex system.

## 7. Product Goals

### Goal 1: Make capture effortless

A user should be able to begin capturing within seconds.

### Goal 2: Produce a useful result quickly

The first meaningful output should appear as soon as practical.

### Goal 3: Understand the full source

Video analysis should include audio and visual content.

### Goal 4: Reduce manual organization

The user should receive one coherent Curated Note.

### Goal 5: Fast-track activation

The user should be able to copy, export, save, or open another app immediately.

### Goal 6: Preserve user ownership

The user should be able to keep their files in their own storage.

### Goal 7: Support future Mission Control

The product should create structured data that can later power the VisionBuilt dashboard.
